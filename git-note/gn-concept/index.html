# 什麼 Git？

「阿就… 版控工具阿？！」 這可能是大部分的人聽到 Git 之後的第一個直覺反應，畢竟業界現在提到版控這個詞，要不是提 Git 就是 SVN 。

不過到底為什麼要用 Git ？我們直接把檔案 Ctrl + A 然後 包起來複製一份，不就控好了嗎？
甚至我們還可以把每個檔案用日期當檔名，就可以知道是哪個時間操作的，很方便吧！

這個方法很傳統也很簡單，可能隨便叫一個剛學會電腦的小學生，都知道可以這麼操作。不過如果我希望在一堆打包的備份檔案裡面，分辨每一包程式到底差異在哪，這可能會是很大的問題，又如果刪了某一包的備份，突然發現那份資料很重要需要復原，可能也有還原不了的風險存在。

透過 Git ，或許很多需要辨別檔案內容、操作檔案內容、更變檔案內容、了解內容版本的問題，可以很方便的解決！

# Git 基本觀念

想像一個城鎮，城鎮中有一個大廣場，廣場中有一個倉庫。
城鎮有許多貨物在運送，這些貨物最後都會被放到倉庫裡頭。而在進入倉庫之前，這些貨物會先被集中到大廣場。
而控管貨物狀態的，就是警衛先生，他會聽從鎮長的指示去整理大廣場中的貨物。
等到鎮長跟他說整理得差不多了，他就會把貨物整批放到倉庫裡面。
有時候鎮長可能覺得廣場的貨物可能少裝了些什麼東西，所以需要從廣場拿走貨物重新包裝，這也會交代給警衛去跟貨物公司溝通處理。
而在廣場之外的貨物，因為鎮長還沒有交代貨物公司送去廣場，所以警衛管不到，這也不是警衛的工作，警衛只會專心的聽從鎮長的指示去處理廣場與倉庫的貨物。

我們把這個想像帶到 Git 的世界中：
鎮長就是使用 Git 的人們，警衛則扮演著 Git 的腳色。
同時整個城鎮就是整包的專案，而貨物則是 Git 所控管 (或是管不到) 的檔案。

廣場外的貨物，就像新增的檔案。它們是尚未被 Git 所追蹤和控制的檔案。 Git 無法對這些檔案進行版本控制，如果刪除或修改這些檔案， Git 無法對其進行恢復。

廣場上的貨物，就像是在暫存區的檔案。這些檔案已經被 Git 追蹤，但尚未被永久地記錄在儲存庫中。暫存區中的檔案表示已經告訴 Git 要將這些檔案納入下一次的提交中。如果檔案在暫存區中被修改或刪除，Git 可以將其還原回最後一次提交的狀態。

倉庫中的貨物，就好比是在儲存庫的檔案。這些檔案已經被 Git 永久地記錄在儲存庫中，形成了不同的版本。這些檔案是穩定的且可以被復原到過去的狀態。

透過上述的說明，我們先簡單總結一個概念：使用 Git 時，可能的狀態包含以下三個

  1. 新增的檔案：尚未被 Git 所追蹤和控制。

    這些檔案就跟尚未加上 Git 版控一樣，Git 管不到他，刪了、改了，Git 想復原也沒辦法

  2. 暫存區（已追蹤）的檔案：已被 Git 追蹤，但尚未被永久地記錄在儲存庫中，表示它們準備納入下一次的提交。

    這些檔案是 Git 暫時追蹤 的檔案，只是讓 Git 知道：哦～我準備要控管你囉～的狀態
    此時如果檔案被修改了，或是被刪除了，Git 都有辦法進行還原

  3. 儲存庫的檔案：已被 Git 永久地記錄在儲存庫中，形成了不同的 版本。

    這些檔案是 Git 已記錄的穩定版檔案，可以解釋成已經定義為一個版本的檔案

而在開發過程的操作，大多會圍繞在 暫存區 中，去控制每一個版本的檔案應該長怎麼樣
在開發完成之後，則會去處理 儲存庫 內的版本狀態

# 常見問題

# Git 跟 GitHub 有什麼差別，還有像是 Git Lab 、TFS 的東西，那是 Git 的別名嗎？

這也是我剛接觸 Git 就優先搞混的東西，看到字樣上都有 Git ，那應該就是一樣的東西吧！
其實 Git 是一個版本控制的 軟體 ，而 GitHub 、 GitLab 這類則是 商業網站
雖然這些網站確實是建構在 Git 之上，提供各種 Git 圖形化介面讓我們更方便的去操作 Git ，不過本質上會是一個 Git 的伺服器。

可以想像成 Git 是我們的電腦本體，我們可以在這上面操作各種檔案
而 GitHub、 GitLab、TFS 這些東西就像 Google drive、One drive 這類東西，是可以讓我們把本機端的資料上傳的雲端空間。
本機的檔案內容跟雲端空間的檔案內容，如果沒特別設定的話，基本上不會互相干擾

事實上， Git 在安裝好之後，即使沒有網路也可以在本機進行檔案的控管
不過如果要把資料 push 到 Git Server ，就如同把檔案上傳到 XXX drive 一樣，是需要透過網路才能執行的！

雖然這些網站很便捷的幫我們包裝好常用到的指令，但由於這些功能都是建構在 Git 之上
如果在不熟悉 Git 的用途下操作，就可能沒辦法把握在網站上操作的動作是否會影響到版本控制。
反過來說，如果希望操作網站能夠更駕輕就熟，那就先建構好基本的 Git 觀念吧！！

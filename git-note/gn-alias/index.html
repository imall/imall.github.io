:::info no-icon
我們在下 Git 指令的時候做的事情幾乎是大同小異，不外乎 git status、git log、git add、git commit -m 等指令不斷的重複操作
打一次可能覺得還好，但如果持續操作的過程，不知不覺中是會浪費很多打字的時間的
常用的指令設定縮寫，下指令時會有種快感，可以節省很多時間！
:::

設定簡寫的指令

而那個設定簡寫的指令長這樣：

git config --global alias.[想要簡寫的名稱] [原本的指令]

常用的簡寫

status

以我最常執行的 git status 來說，可以這樣設定簡寫

git config --global alias.st status

設定完之後，當需要用到 git status 的場景，只要執行這串，就會有git status的功能

git st

log

這篇文章 提到查詢 log 的方式，有說到一些參數能處理 log 的顯示，這些參數如果沒有常使用，很難在需要的時候去記憶。
如果使用簡寫的話，就連記都不用去記了：

設為一行、顯示基本圖表

git config --global alias.l  "log --oneline --graph"

實務上我很少直接使用 git log 指令，因為螢幕不夠大顯示的資料不夠精簡，導致能一次顯示的內容太少，大多數的情況下都會加上 --oneline --graph參數
設定好上面指令後，我只要執行 git l 就可以看到想要的效果

甚至可以設定一些 format 的 log 訊息讓資訊更精美：

顯示 format 資訊

git config --global alias.lg "log --color --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"

不要問我這指令的內容是什麼，我也是抄來的

設定好之後執行 git lg 就會跑出這樣的格式內容

文章中也有提到，當有其他 Git GUI 可以查看線圖的時候，git log 系列指令真的變得很少用到了

add

啥？ git add 也要簡寫？對，我就懶！

將更動的所有內容都加到暫存區

我設定了這個指令取代 add . 或 add --all，我就不用多去打 . 或是 --all 了

git config --global alias.all  "add --all"

將檔案內容移出暫存區

在不知道 git reset 也有移除暫存區功能前，我始終記不起來 restore --staged ，所以設定了這個：

git config --global alias.rmadd "restore --staged"

以rm開頭，代表remove的概念，不過自從知道 reset 也能做到一樣的功能時，這指令就很少用了 ~~字數一樣是要簡寫個毛 XD ~~

commit

commit 系列就精彩了，除了基本的 commit 之外，一堆修改內容的指令都屬於 commit 這類
紀錄一下我設定了哪些東西：

commit 本身的簡寫

前面說道，很多動作都是需要 commit 開頭，於是先把 commit 設定一個縮寫

git config --global alias.cm commit

commit message

最基本的提交版本簡寫，應該來一下的

git config --global alias.cmm "commit -m"

命名概念很單純，就是 commit + -m = cmm 而已

git add + git commit

當不想先 git add 再 git commit 的話 git commit -a -m 會是個方便的指令

但懶還要更懶！ 來設定這樣的縮寫

git config --global alias.cmam "commit -am"

更改上一版 commit 的內容

git config --global alias.cmcm "commit --amend --no-edit"

這算是一個蠻常用的指令，在還沒 push 之前，透過這個指令來修改上一個版本內容
由於參數有點多，我就兩個 commit ： commit + commit 這個概念，代表要對上一次的內容進行更動

更改上一版 commit 的訊息

git config --global alias.cmrn "commit --amend -m"

當打錯字的時候，用這個指令來改訊息頗為方便
我的命名概念是 commit + rename = cmrn

branch

branch 本身的縮寫

git config --global alias.br branch

branch 本身就有查看分支的用途，對他設定縮寫合情合理

重新命名目前分支

git config --global alias.brm "branch -m"

這個指令也算是很常用的，分支就是個怎麼改名都不會影響專案內容的東西XD

刪除分支

git config --global alias.brd "branch -d"

強制建立分支

git config --global alias.brf "branch -f"

這個指令我比較多用在把已存在的分支強制移動到最新的分支之上，也算是個好用的指令

checkout

checkout 本身的縮寫

git config --global alias.co checkout

建立並切換分支

git config --global alias.cob "checkout -b"

相信大部分的人新增分支都會的下一步都會直接切換過去，這個指令可以讓速度快還要更快！

reset

reset 本身的縮寫

git config --global alias.r reset

可能會有人覺得除了 reset 之外 ，還有 rebase 、 revert 都是 r 開頭的指令
不過我認為這其中，reset 屬於最常用到的，把 r 作為 reset 的代表是可以的。但這就自由心證囉

reset 的 hard 模式

git config --global alias.rh "reset --hard"

reset 指令很多情況幾乎都是要用 hard 模式，於是我設定了這個縮寫

還原所有檔案

git config --global alias.rall "reset --hard HEAD"

我很常會使用 git rest --hard HEAD 指令還原到上一版的原始狀態，乾脆設一個縮寫。

把設定好的縮寫刪掉

當發現有些縮寫設定後卻都用不到，可以下這個指令把縮寫移除

git config --global --unset alias.[縮寫名稱]

除此之外，也可以直接去查看.gitconfig檔案把縮寫刪除

查看設定了哪些指令

透過指令查詢

要查看自己設定了哪些縮寫的話，可以這樣下指令查詢：

git config --get-regexp alias

這個指令我常常記不起來，乾脆對這個指令也下一個縮寫

git config --global alias.getalias "config --get-regexp alias"

到.gitconfig查詢

  1. Windows 系統使用者，要到這個路徑找.gitconfig檔案： C:\Users\[你的名字]

  2. Mac 使用者則是到終端機執行 open ~/.gitconfig 指令開啟

找到 [alias] 區塊，就可以看到設定的所有資訊：

實際上只要按照這個格式去設定、刪除，就可以做到上面指令的效果，當知道這件事情後，也不太需要去下指令設定縮寫了。
